# DRY-аудит фронтенду — 2026-04-22

Документ фіксує дублювання коду/JSX/даних у фронтенді Stash. Для кожної знахідки — де саме повторюється, що запропоновано винести, куди покласти, і пріоритет. Джерела перевірені через grep на момент написання.

Структура папок-цілей:

- `src/shared/ui/` — React-примітиви (вже існує).
- `src/shared/hooks/` — React-хуки (вже існує: `useKeyboardNav`, `useSuppressibleConfirm`).
- `src/shared/format/` — **створити**: форматери (bytes, duration, date).
- `src/shared/util/` — **створити**: утиліти (clipboard, reveal, toast helpers).

Пріоритети: **P0** — обов'язково (3+ копії або розбіжність у поведінці); **P1** — бажано (2 копії); **P2** — для майбутньої консистентності.

## 7. Async-load pattern (useState + useEffect + mountedRef)

**P1.** Майже в кожному `*Panel.tsx` системного модуля повторюється:

```tsx
const [rows, setRows] = useState<T[] | null>(null);
const [error, setError] = useState<string | null>(null);
const mountedRef = useRef(true);
const load = useCallback(async () => {
  try { const d = await api(); if (!mountedRef.current) return; setRows(d); setError(null); }
  catch (e) { if (!mountedRef.current) return; setError(String(e)); }
}, []);
useEffect(() => { load(); }, [load]);
```

**Файли:** `ProcessesPanel`, `NetworkPanel`, `CachesPanel`, `DiskHogsPanel`, `NodeModulesPanel`, `LargeFilesPanel`, `TrashBinsPanel`, `DuplicatesPanel`, `PrivacyPanel`, `LaunchAgentsPanel` — ~10 панелей.

**Фікс:**

- `src/shared/hooks/useAsync.ts`:
  ```ts
  export function useAsync<T>(fn: () => Promise<T>, deps?: React.DependencyList): {
    data: T | null; error: string | null; loading: boolean; reload: () => Promise<void>;
  }
  ```
- Пересадити панелі (10 файлів, мінус ~8 рядків boilerplate у кожному).
