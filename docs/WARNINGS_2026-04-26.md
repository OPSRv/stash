## 🟡 Rust warnings (release build, не блокери але "cargo fix" їх прибере)

### `src-tauri/src/modules/music/commands.rs:123`
```
warning: variable does not need to be mutable
   |
123|     let mut builder = tauri::webview::WebviewBuilder::new(LABEL, WebviewUrl::External(url))
   |         ----^^^^^^^
```
**Fix**: умовно `mut` тільки в `#[cfg(debug_assertions)]` гілці що далі викликає `builder = builder.devtools(true)`. Прибрати `mut` з зовнішнього `let` і додати `let mut builder` всередину `#[cfg(debug_assertions)]` блоку — або обернути все в `let builder = { ... if cfg!(debug_assertions) { ... } ... };`.

### `src-tauri/src/modules/webchat/commands.rs:326`
```
warning: variable does not need to be mutable
   |
326|     let mut builder = tauri::webview::WebviewBuilder::new(&label, WebviewUrl::External(parsed))
   |         ----^^^^^^^
```
**Fix**: ідентично music. Те саме `WebviewBuilder` + `#[cfg(debug_assertions)] { builder = builder.devtools(true); }`.

> Обидва генеруються по 2 рази в release build (універсальний бандл компілює `aarch64-apple-darwin` + `x86_64-apple-darwin`), сумарно `cargo` каже `generated 2 warnings`. `cargo fix --lib -p stash-app --release` зробить fix автоматично.

## 🟡 Frontend warnings

### CSS: invalid empty selector у `src/styles/tokens.css:405-408`
```
src/styles/tokens.css:
  /// Slightly deeper tint for list surfaces where the active row needs extra
  /// contrast against a busier neighbour column (e.g. the Notes list sitting
  /// beside the editor). Opt-in via the `row-active-strong` modifier.
  .row-active-strong.row-active {
```
**Fix**: `///` — невалідний CSS-коментар (rustdoc-style попав у `.css`). Lightning CSS трактує його як початок селектора → "Invalid empty selector". Замінити на `/* ... */`. Також глянь решту файла — `grep -n "^  ///" src/styles/tokens.css` показує ще такі ж рядки.
